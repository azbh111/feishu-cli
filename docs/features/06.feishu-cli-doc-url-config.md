# 06. 文档链接前缀可配置化（DocURL）

- **分类**: feishu-cli
- **提交官方**: 是
- **类型**: feat
- **PR 分支**: `feat/configurable-doc-url`
- **PR 标题**: `feat(config): 支持自定义文档链接前缀（DocURL）`

## 变更文件

| 文件 | 改动说明 |
|------|---------|
| `internal/config/config.go` | 新增 `DocURL` 字段（默认 `https://feishu.cn`，环境变量 `FEISHU_DOC_URL`） |
| `cmd/create_document.go` | 硬编码 `https://feishu.cn` → `config.Get().DocURL` |
| `cmd/get_document.go` | 同上 |
| `cmd/import_markdown.go` | 同上 |

## 问题描述

文档链接前缀硬编码为 `https://feishu.cn`，无法适配 `larkoffice.com`、`larksuite.com` 等不同飞书域名。不同企业的飞书域名可能不同。

## 改动内容

- `config.go` 新增 `DocURL` 配置项，支持配置文件 `doc_url` 和环境变量 `FEISHU_DOC_URL`
- 3 个 cmd 文件将硬编码替换为 `config.Get().DocURL`

## 依赖

无。

## Cherry-pick 方式

```bash
git diff main..dev -- internal/config/config.go  # 提取 DocURL 相关 hunk（排除 FolderID）
git diff main..dev -- cmd/create_document.go | git apply
git diff main..dev -- cmd/get_document.go | git apply
# import_markdown.go 中只取 DocURL 替换的 hunk
```

## 验证

```bash
go test ./internal/config/...
go build -o feishu-cli .
./feishu-cli doc create --title "测试"  # 检查输出链接前缀
```
