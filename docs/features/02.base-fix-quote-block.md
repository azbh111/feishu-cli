## 问题描述

三个独立 bug，均与引用块（QuoteContainer）相关：

1. **lazy continuation 吞文本**：goldmark 的 lazy continuation 会将引用块后紧跟的非引用行视为引用块一部分。例如：
   ```markdown
   > 引用
   这行被错误地解析进引用块
   ```
2. **多段落空行丢失**：`> 段落1\n>\n> 段落2` 中间的空行在导入时丢失
3. **导出空行未还原**：导出时跳过了所有空文本子块，导致多段落引用块的段落间空行消失

## 改动内容

- 导入侧：`normalizeBlockquoteEnding` 在引用块末尾插入空行终止 lazy continuation；`convertBlockquote` 用计数器在段落间插入空 Text 块
- 导出侧：`convertQuoteContainerWithDepth` 跳过开头空行，保留中间空行输出 `>`
