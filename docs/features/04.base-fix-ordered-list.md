## 问题描述

4 个问题：

1. **所有有序列表项输出 `1.`**：导出时固定输出 `1.`，丢失了真实编号信息
2. **自定义起始编号不支持**：`3. 第三项\n4. 第四项` 导入后 Sequence 字段为空
3. **嵌套有序列表序号污染**：嵌套子列表的序号和父级列表共享计数器，导致编号错乱
4. **嵌套缩进 2 空格**：标准 Markdown 需要 4 空格表示嵌套层级，2 空格在部分解析器下不识别为嵌套

## 改动内容

### 导入侧
- `convertList` 读取 `node.Start`，当 > 1 时将起始编号传递给 `convertListItem`
- `convertListItem` 新增 `seq` 参数，设置 `TextStyle.Sequence`

### 导出侧
- 新增 `orderedSeq` 字段，连续 Ordered 块自增，类型切换时重置
- `convertOrdered` 输出真实编号（读取 Sequence 覆盖或使用自增值）
- `convertBullet`、`convertOrdered`、`convertTodoWithDepth` 中嵌套子列表使用独立 `childOrderedSeq` 计数器
- `strings.Repeat("  ", indent)` → `strings.Repeat("    ", indent)`（2→4 空格）
- 列表块不再追加多余 `\n`
