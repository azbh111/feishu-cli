# 08. Mermaid 导入改用文本绘图块

- **分类**: feishu-cli
- **提交官方**: 是
- **类型**: feat
- **PR 分支**: `feat/mermaid-text-drawing`
- **PR 标题**: `feat(import): Mermaid 图表改用 TextDrawing 块导入`

## 变更文件

| 文件 | 改动说明 |
|------|---------|
| `internal/client/docx.go` | 新增 `blockTypeAddOns`、`componentTypeTextDrawing` 常量；新增 `AddTextDrawing()` 方法；新增 `jsonEscapeString()`、`strPtr()` 辅助函数 |
| `cmd/import_markdown.go` | Mermaid 图表从画板 API 改为调用 `AddTextDrawing`（block_type=40 文本绘图块） |

## 问题描述

之前 Mermaid 图表通过画板 API 导入（创建画板块 → 导入 Mermaid 到画板），存在问题：
- 画板 API 不稳定，大量 Mermaid 语法飞书无法解析
- 画板块无法原地编辑 Mermaid 源码
- 失败率较高，需要复杂的重试和降级逻辑

飞书支持 TextDrawing（文本绘图，block_type=40）块类型，原生渲染 Mermaid，支持在文档内直接编辑源码。

## 改动内容

- `client/docx.go`：新增 `AddTextDrawing` 方法，通过 `CreateBlock` API 创建 AddOns 块（type=40），内含 TextDrawing 组件
- `import_markdown.go`：Mermaid 代码块识别后调用 `AddTextDrawing` 替代原画板导入流程

## 依赖

无。但 PlantUML 仍保持画板导入方式不变。

## Cherry-pick 方式

```bash
# client/docx.go 中提取 AddTextDrawing 相关代码
# import_markdown.go 中提取 Mermaid 处理逻辑的改动 hunk
```

**注意**：import_markdown.go 中 Mermaid 改动可能与三阶段管道的 diagram worker 逻辑交叉，需仔细确认影响范围。建议同步更新 CLAUDE.md 的「核心功能」章节说明。

