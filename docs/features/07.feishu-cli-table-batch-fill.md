# 07. 表格填充批量优化

- **分类**: feishu-cli
- **提交官方**: 是
- **类型**: perf
- **PR 分支**: `perf/table-batch-fill`
- **PR 标题**: `perf(client): 表格单元格批量填充，减少 API 调用`

## 变更文件

| 文件 | 改动说明 |
|------|---------|
| `internal/client/docx.go` | `fillTableCellsInternal` 重构：单块 cell 合并为一次 BatchUpdateBlocks；新增 `FillTableCellsCounted` / `FillTableCellsRichCounted` 返回 API 调用次数 |

## 问题描述

原来的表格填充逻辑对每个单元格单独调用 API，N 个单元格 = N 次 API 请求。大表格（如 9x9）导入很慢且容易触发频率限制。

## 改动内容

- **单块 cell 批量合并**：将所有只含单个文本块的单元格合并为一次 `BatchUpdateBlocks` API 调用
- **多块 cell 逐个处理**：单元格内含多个块的仍逐个处理（API 限制）
- **失败降级**：批量失败时自动降级为逐个更新
- **Counted 变体**：`FillTableCellsCounted` / `FillTableCellsRichCounted` 返回实际 API 调用次数，用于 `--verbose` 统计输出
